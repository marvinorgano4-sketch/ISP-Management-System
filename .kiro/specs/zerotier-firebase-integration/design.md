# Design Document: ZeroTier Remote Access at Firebase Cloud Database Integration

## Overview

This design document outlines the technical approach for integrating ZeroTier VPN for secure remote access and migrating from SQLite to Firebase Firestore cloud database in the existing ISP Billing System. The integration maintains backward compatibility with existing functionality while adding cloud-based data storage and secure remote access capabilities.

The system will continue to use Flask as the web framework, but will replace SQLAlchemy/SQLite with Firebase Admin SDK for database operations. ZeroTier will provide a secure virtual private network layer that allows administrators to access the billing system from any location without exposing the application to the public internet.

## ZeroTier Network Design

### Network Topology

The ZeroTier network will use a **star topology** with the server as the central node:

- **Network Type**: Private network with manual authorization
- **IP Range**: 10.147.x.x/16 (auto-assigned by ZeroTier)
- **Access Control**: Whitelist-based (manual device approval)
- **Encryption**: AES-256 end-to-end encryption

### ZeroTier Configuration

**Server Configuration:**
```yaml
Network ID: [Generated by ZeroTier Central]
Server Virtual IP: 10.147.17.1 (example)
Flask Bind Address: 0.0.0.0:5000
Allowed Devices: Manually authorized admin devices
```

**Client Configuration:**
```yaml
Network ID: [Same as server]
Client Virtual IP: 10.147.17.x (auto-assigned)
Connection Method: Join network via ZeroTier One client
Access URL: http://10.147.17.1:5000
```

### Connection Flow

1. **Initial Setup:**
   - Admin installs ZeroTier One on server
   - Creates private network via ZeroTier Central
   - Obtains Network ID
   - Server joins network and receives virtual IP

2. **Client Connection:**
   - Admin installs ZeroTier One on device
   - Joins network using Network ID
   - Waits for authorization from network admin
   - Once authorized, receives virtual IP
   - Accesses Flask app via server's virtual IP

3. **Security Measures:**
   - All traffic encrypted with AES-256
   - Certificate pinning for ZeroTier controller
   - No public internet exposure of Flask app
   - Firewall rules restrict access to ZeroTier interface only

### Flask Application Binding

The Flask application will bind to all interfaces (`0.0.0.0`) but will be protected by firewall rules that only allow traffic from:
- Local network (existing functionality)
- ZeroTier virtual interface (new functionality)

Configuration in `app.py`:
```python
if __name__ == '__main__':
    app = create_app()
    # Bind to all interfaces, firewall controls access
    app.run(debug=False, host='0.0.0.0', port=5000)
```

### Firestore Data Model

Firebase Firestore uses a document-based NoSQL structure. We'll organize data into collections that mirror the existing SQLite tables:


**Collection Structure:**


**Key Design Decisions:**

1. **Document IDs**: Use Firestore auto-generated IDs instead of integer IDs
2. **References**: Store related document IDs as strings (e.g., `client_id` in billings)
3. **Timestamps**: Use Firestore `timestamp` type for all datetime fields
4. **Indexes**: Create composite indexes for common queries (e.g., client status + name)
5. **Denormalization**: Store frequently accessed data redundantly to reduce reads

### Firebase Admin SDK Integration

**Initialization:**

The Firebase Admin SDK will be initialized at application startup using a service account key file:

```python
# firebase_config.py
import firebase_admin
from firebase_admin import credentials, firestore
import os

def initialize_firebase():
    """Initialize Firebase Admin SDK"""
    cred_path = os.getenv('FIREBASE_CREDENTIALS_PATH', 'firebase-credentials.json')
    
    if not os.path.exists(cred_path):
        raise FileNotFoundError(f"Firebase credentials not found at {cred_path}")
    
    cred = credentials.Certificate(cred_path)
    firebase_admin.initialize_app(cred)
    
    return firestore.client()
```

**Service Account Credentials:**

The service account JSON file will contain:
- Project ID
- Private key
- Client email
- Authentication tokens

This file must be:
- Stored securely outside version control
- Readable only by the Flask application user
- Backed up in a secure location

### Firebase Security Rules

Firestore security rules will enforce server-side access control:

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // Only allow access from authenticated Firebase Admin SDK
    match /{document=**} {
      allow read, write: if false;
    }
  }
}
```

Since we're using Firebase Admin SDK (not client SDK), all access control will be handled at the application level. The security rules deny all direct client access, ensuring only the Flask application can interact with Firestore.

### Data Transformation Rules

**SQLite to Firestore Mapping:**

| SQLite Type | Firestore Type | Transformation |
|-------------|----------------|----------------|
| INTEGER (ID) | Auto-generated string | Generate new Firestore ID, maintain mapping |
| TEXT | string | Direct copy |
| REAL/FLOAT | number | Direct copy |
| DATETIME | timestamp | Convert to Firestore timestamp |
| BOOLEAN | boolean | Direct copy |

**Relationship Handling:**

1. **Foreign Keys**: Convert integer foreign keys to Firestore document ID strings
2. **ID Mapping**: Maintain a mapping table during migration (old_id → new_id)
3. **References**: Update all references after initial data import

**Example Transformation:**

```python
# SQLite Client record

    'created_at': '2024-01-15 10:30:00'
}

    'full_name': 'Juan Dela Cruz',
    'created_at': firestore.SERVER_TIMESTAMP,
    # id is auto-generated by Firestore

# Store mapping: sqlite_id=1 → firestore_id='abc123xyz'
```

### Migration Validation

**Pre-Migration Checks:**
- Verify SQLite database exists and is readable
- Verify Firebase credentials are valid
- Check network connectivity to Firebase
- Verify sufficient Firestore quota

**Post-Migration Validation:**
- Count records in each collection matches SQLite tables
- Verify all relationships are intact
- Spot-check random records for data accuracy
- Verify indexes are created
- Test query performance

**Validation Queries:**

```python
def validate_migration(self):
    """Validate migration success"""
    validations = []
    
    # Count validation
    sqlite_user_count = self.sqlite_conn.execute("SELECT COUNT(*) FROM users").fetchone()[0]
    firestore_user_count = len(self.firestore_db.collection('users').get())
    validations.append(sqlite_user_count == firestore_user_count)
    
    # Relationship validation
    # Check that all billing.client_id references exist in clients
    billings = self.firestore_db.collection('billings').get()
    for billing in billings:
        client_exists = self.firestore_db.collection('clients').document(billing.get('client_id')).get().exists
        validations.append(client_exists)
    
    return all(validations)
```

### Rollback Strategy

**Rollback Approach:**

1. **Backup Before Migration**: Create SQLite backup file
2. **Transaction Log**: Log all Firestore writes during migration
3. **Rollback Execution**: Delete all created Firestore documents if migration fails
4. **Restore SQLite**: Keep SQLite database intact until migration verified

**Rollback Implementation:**

```python
def rollback(self):
    """Rollback migration by deleting all created documents"""
    print("Rolling back migration...")
    
    # Delete in reverse order to handle references
    collections = ['receipts', 'payments', 'billings', 'clients', 'users']
    
    for collection_name in collections:
        docs = self.firestore_db.collection(collection_name).get()
        for doc in docs:
            doc.reference.delete()
            print(f"Deleted {collection_name}/{doc.id}")
    
    print("Rollback complete. SQLite database unchanged.")
```

## Service Layer Architecture

### FirebaseService Base Class

A base service class will provide common CRUD operations for all models:

```python
# services/firebase_service.py

from firebase_admin import firestore
from datetime import datetime
from typing import List, Dict, Optional, Any

class FirebaseService:
    """Base service class for Firebase Firestore operations"""
    
    def __init__(self, db, collection_name):
        self.db = db
        self.collection_name = collection_name
        self.collection = db.collection(collection_name)
    
    def create(self, data: Dict[str, Any]) -> str:
        """Create a new document"""
        data['created_at'] = firestore.SERVER_TIMESTAMP
        doc_ref = self.collection.document()
        doc_ref.set(data)
        return doc_ref.id
    
    def get(self, doc_id: str) -> Optional[Dict[str, Any]]:
        """Get a document by ID"""
        doc = self.collection.document(doc_id).get()
        if doc.exists:
            data = doc.to_dict()
            data['id'] = doc.id
            return data
        return None
    
    def get_all(self, filters: Optional[Dict] = None, order_by: Optional[str] = None) -> List[Dict[str, Any]]:
        """Get all documents with optional filters"""
        query = self.collection
        
        if filters:
            for field, value in filters.items():
                query = query.where(field, '==', value)
        
        if order_by:
            query = query.order_by(order_by)
        
        docs = query.stream()
        results = []
        for doc in docs:
            data = doc.to_dict()
            data['id'] = doc.id
            results.append(data)
        
        return results
    
    def update(self, doc_id: str, data: Dict[str, Any]) -> bool:
        """Update a document"""
        data['updated_at'] = firestore.SERVER_TIMESTAMP
        doc_ref = self.collection.document(doc_id)
        
        if not doc_ref.get().exists:
            return False
        
        doc_ref.update(data)
        return True
    
    def delete(self, doc_id: str) -> bool:
        """Delete a document"""
        doc_ref = self.collection.document(doc_id)
        
        if not doc_ref.get().exists:
            return False
        
        doc_ref.delete()
        return True
    
    def query(self, field: str, operator: str, value: Any) -> List[Dict[str, Any]]:
        """Execute a custom query"""
        docs = self.collection.where(field, operator, value).stream()
        results = []
        for doc in docs:
            data = doc.to_dict()
            data['id'] = doc.id
            results.append(data)
        
        return results
```

### Updated Service Classes

Each existing service class will be updated to use FirebaseService instead of SQLAlchemy:

**UserService (Firebase version):**

```python
# services/user_service.py

from services.firebase_service import FirebaseService
import bcrypt

class UserService:
    """Service for user authentication and management"""
    
    def __init__(self, db):
        self.firebase = FirebaseService(db, 'users')
    
    def create_user(self, username: str, password: str, full_name: str) -> str:
        """Create a new user"""
        # Check if username exists
        existing = self.firebase.query('username', '==', username)
        if existing:
            raise ValueError('Username already exists')
        
        # Hash password
        password_hash = bcrypt.hashpw(password.encode('utf-8'), bcrypt.gensalt()).decode('utf-8')
        
        # Create user
        user_data = {
            'username': username,
            'password_hash': password_hash,
            'full_name': full_name,
            'last_login': None
        }
        
        return self.firebase.create(user_data)
    
    def authenticate(self, username: str, password: str) -> Optional[Dict]:
        """Authenticate user"""
        users = self.firebase.query('username', '==', username)
        
        if not users:
            return None
        
        user = users[0]
        
        # Verify password
        if bcrypt.checkpw(password.encode('utf-8'), user['password_hash'].encode('utf-8')):
            # Update last login
            self.firebase.update(user['id'], {'last_login': firestore.SERVER_TIMESTAMP})
            return user
        
        return None
```

**ClientService (Firebase version):**

```python
# services/client_service.py

from services.firebase_service import FirebaseService
from typing import List, Dict, Optional

class ClientService:
    """Service for client management"""
    
    def __init__(self, db):
        self.firebase = FirebaseService(db, 'clients')
    
    def create_client(self, data: Dict) -> str:
        """Create a new client"""
        # Validate PPPoE username uniqueness
        existing = self.firebase.query('pppoe_username', '==', data['pppoe_username'])
        if existing:
            raise ValueError('PPPoE username already exists')
        
        # Validate required fields
        required = ['full_name', 'pppoe_username', 'plan_name', 'plan_amount']
        for field in required:
            if field not in data:
                raise ValueError(f'Field {field} is required')
        
        # Set defaults
        data.setdefault('status', 'active')
        data.setdefault('address', '')
        data.setdefault('contact_number', '')
        data.setdefault('email', '')
        
        return self.firebase.create(data)
    
    def get_client(self, client_id: str) -> Optional[Dict]:
        """Get client by ID"""
        return self.firebase.get(client_id)
    
    def get_all_clients(self, filters: Optional[Dict] = None) -> List[Dict]:
        """Get all clients with optional filters"""
        return self.firebase.get_all(filters, order_by='full_name')
    
    def update_client(self, client_id: str, data: Dict) -> bool:
        """Update client"""
        # Validate PPPoE username if changed
        if 'pppoe_username' in data:
            existing = self.firebase.query('pppoe_username', '==', data['pppoe_username'])
            # Filter out current client
            existing = [c for c in existing if c['id'] != client_id]
            if existing:
                raise ValueError('PPPoE username already exists')
        
        return self.firebase.update(client_id, data)
    
    def search_clients(self, query: str) -> List[Dict]:
        """Search clients by name or username"""
        # Note: Firestore doesn't support LIKE queries
        # We'll need to implement client-side filtering or use Algolia/Elasticsearch
        all_clients = self.firebase.get_all()
        query_lower = query.lower()
        
        results = [
            client for client in all_clients
            if query_lower in client.get('full_name', '').lower()
            or query_lower in client.get('pppoe_username', '').lower()
            or query_lower in client.get('contact_number', '').lower()
        ]
        
        return results
```

### Real-Time Listener Implementation

Firebase Firestore supports real-time listeners that notify the application when data changes:

```python
# services/realtime_service.py

from firebase_admin import firestore

class RealtimeService:
    """Service for handling real-time Firestore updates"""
    
    def __init__(self, db):
        self.db = db
        self.listeners = {}
    
    def listen_to_collection(self, collection_name: str, callback):
        """Listen to changes in a collection"""
        def on_snapshot(col_snapshot, changes, read_time):
            for change in changes:
                if change.type.name == 'ADDED':
                    callback('added', change.document.to_dict())
                elif change.type.name == 'MODIFIED':
                    callback('modified', change.document.to_dict())
                elif change.type.name == 'REMOVED':
                    callback('removed', change.document.to_dict())
        
        # Create listener
        col_ref = self.db.collection(collection_name)
        listener = col_ref.on_snapshot(on_snapshot)
        
        # Store listener reference
        self.listeners[collection_name] = listener
        
        return listener
    
    def stop_listening(self, collection_name: str):
        """Stop listening to a collection"""
        if collection_name in self.listeners:
            self.listeners[collection_name].unsubscribe()
            del self.listeners[collection_name]
```

**Usage in Flask Application:**


### Security Best Practices

1. **Credentials Storage:**
   - Never commit credentials to version control
   - Use `.gitignore` to exclude `.env` and `firebase-credentials.json`
   - Store credentials in secure location with restricted permissions (chmod 600)

2. **Firebase Service Account:**
   - Use dedicated service account for the application
   - Grant minimum required permissions (Firestore read/write only)
   - Rotate service account keys periodically

3. **ZeroTier Security:**
   - Use private network (not public)
   - Enable manual device authorization
   - Regularly audit authorized devices
   - Disable unused devices immediately

4. **Flask Security:**
   - Use strong SECRET_KEY (minimum 32 random bytes)
   - Disable debug mode in production
   - Enable CSRF protection
   - Use secure session cookies

## Error Handling and Monitoring

### Connection Failure Handling

**Firebase Connection Errors:**

```python
# services/firebase_error_handler.py

from firebase_admin import firestore
from functools import wraps
import time
import logging

logger = logging.getLogger(__name__)

def retry_on_failure(max_retries=3, backoff_factor=2):
    """Decorator to retry Firebase operations on failure"""
    def decorator(func):
        @wraps(func)
        def wrapper(*args, **kwargs):
            retries = 0
            while retries < max_retries:
                try:
                    return func(*args, **kwargs)
                except Exception as e:
                    retries += 1
                    if retries >= max_retries:
                        logger.error(f"Firebase operation failed after {max_retries} retries: {e}")
                        raise
                    
                    wait_time = backoff_factor ** retries
                    logger.warning(f"Firebase operation failed, retrying in {wait_time}s: {e}")
                    time.sleep(wait_time)
        
        return wrapper
    return decorator

class FirebaseErrorHandler:
    """Handle Firebase-specific errors"""
    
    @staticmethod
    def handle_error(error):
        """Convert Firebase errors to user-friendly messages"""
        error_messages = {
```

```python
import subprocess
import logging

logger = logging.getLogger(__name__)

class ZeroTierMonitor:
    """Monitor ZeroTier connection status"""
    
    @staticmethod
    def check_connection(network_id):
        """Check if connected to ZeroTier network"""
        try:
            result = subprocess.run(
                ['zerotier-cli', 'listnetworks'],
                capture_output=True,
                text=True,
                timeout=5
            )
            
            if result.returncode == 0:
                # Check if network_id is in output and status is OK
                for line in result.stdout.split('\n'):
                    if network_id in line and 'OK' in line:
                        return True
            
            return False
        except Exception as e:
            logger.error(f"Failed to check ZeroTier connection: {e}")
            return False
    
    @staticmethod
    def get_virtual_ip(network_id):
        """Get assigned virtual IP address"""
        try:
            result = subprocess.run(
                ['zerotier-cli', 'listnetworks'],
                capture_output=True,
                text=True,
                timeout=5
            )
            
            if result.returncode == 0:
                for line in result.stdout.split('\n'):
                    if network_id in line:
                        # Parse IP from output
                        parts = line.split()
                        for part in parts:
                            if '/' in part:  # IP with CIDR notation
                                return part.split('/')[0]
            
            return None
        except Exception as e:
            logger.error(f"Failed to get ZeroTier IP: {e}")
            return None
```

```python
# services/performance_monitor.py

import time
import logging
from functools import wraps

logger = logging.getLogger('performance')

def track_performance(operation_name):
    """Decorator to track operation performance"""
    def decorator(func):
        @wraps(func)
        def wrapper(*args, **kwargs):
            start_time = time.time()
            try:
                result = func(*args, **kwargs)
                elapsed = time.time() - start_time
                logger.info(f"{operation_name} completed in {elapsed:.3f}s")
                return result
            except Exception as e:
                elapsed = time.time() - start_time
                logger.error(f"{operation_name} failed after {elapsed:.3f}s: {e}")
                raise
        return wrapper
    return decorator

# Usage
@track_performance("Get all clients")

    return self.firebase.get_all()
```

## Correctness Properties

A property is a characteristic or behavior that should hold true across all valid executions of a system—essentially, a formal statement about what the system should do. Properties serve as the bridge between human-readable specifications and machine-verifiable correctness guarantees.

### Property 1: Migration Record Count Preservation

For any SQLite table (users, clients, billings, payments, receipts), the number of records migrated to Firestore should equal the number of records in the source SQLite table.

**Validates: Requirements 5.1, 5.2, 5.3, 5.4, 5.5**

### Property 2: Migration Data Integrity

For any record migrated from SQLite to Firestore, all field values and relationships should be preserved exactly, with only the ID format changing from integer to Firestore document ID.

**Validates: Requirements 5.6**

### Property 3: Migration Rollback on Error

For any error that occurs during migration, the system should rollback all Firestore changes and report detailed error information, leaving the SQLite database unchanged.

**Validates: Requirements 5.8**

### Property 4: Firebase CRUD Round-Trip

For any valid data model (User, Client, Billing, Payment, Receipt), creating a record then immediately reading it should return data equivalent to what was created.

**Validates: Requirements 6.1, 6.2**

### Property 5: Firebase Update Persistence

For any existing Firestore record, updating specific fields should result in those fields being modified in the database while other fields remain unchanged.

**Validates: Requirements 6.3**

### Property 6: Firebase Delete Removal

For any existing Firestore record, deleting it should result in the record no longer being retrievable from the database.

**Validates: Requirements 6.4**

### Property 7: Firebase Query Filter Correctness

**Validates: Requirements 6.7**

### Property 8: Password Hashing Security

**Validates: Requirements 7.3**

### Property 9: Authentication Credential Verification

**Validates: Requirements 7.1**

### Property 10: User Registration Storage

## Error Handling

### Firebase Connection Errors

The system will handle Firebase connection errors using a retry mechanism with exponential backoff:

1. **Initial Failure**: Log error and wait 2 seconds
2. **Second Failure**: Log error and wait 4 seconds
3. **Third Failure**: Log error and wait 8 seconds
4. **Final Failure**: Log error, display user-friendly message, and raise exception

Error messages will be translated to Filipino for user display:
- `PERMISSION_DENIED`: "Hindi ka may permiso na mag-access ng data na ito."
- `NOT_FOUND`: "Hindi nahanap ang hinihinging data."
- `UNAVAILABLE`: "Hindi available ang Firebase service. Subukan ulit mamaya."

### ZeroTier Connection Errors

The system will monitor ZeroTier connection status and handle failures:

1. **Connection Check**: Verify ZeroTier network status at startup
2. **IP Verification**: Confirm virtual IP assignment
3. **Periodic Monitoring**: Check connection status every 5 minutes
4. **Failure Handling**: Log detailed error and display troubleshooting steps

### Migration Errors

The migration tool will handle errors with automatic rollback:

1. **Validation Errors**: Check data integrity before migration
2. **Write Errors**: Rollback all Firestore writes on any failure
3. **Relationship Errors**: Verify all foreign key references exist
4. **Logging**: Record all migration steps for debugging

### Application Startup Errors

The system will validate all configuration at startup:

1. **Configuration Validation**: Check all required environment variables
2. **Firebase Initialization**: Verify credentials and connection
3. **ZeroTier Status**: Check network connection status
4. **Graceful Failure**: Display clear error messages with correction steps
